## 0. 環境構築・ツール編

### 0-1. node_modulesとGit管理
**課題意図**:
`node_modules` フォルダを Git にコミットすべきかどうか、およびその理由と対処法を学ぶ。

**回答コード例**:
プロジェクトルートに `.gitignore` ファイルを作成し、以下を記述して `node_modules` を除外します。

```gitignore
node_modules/
dist/
```

**解説**:
- `node_modules` は依存ライブラリの集合体でサイズが非常に大きいです。
- `package.json` と `package-lock.json` があれば、`npm install` コマンドで誰でも同じ環境を再現できるため、Git で共有する必要がありません。
- `.gitignore` に記述することで、Git の管理対象から除外され、リポジトリを軽量に保てます。


## 1. 基礎編：Vue.jsの基本操作

### 1-1. カウンター機能の実装
**課題意図**:
Vue.js の基本である「リアクティブなデータ（状態）」と「イベントハンドリング」を理解する。

**回答コード例**:
```vue
<script setup>
import { ref } from 'vue'

const count = ref(0) // リアクティブな変数を定義

const increment = () => {
    count.value++ // 値を更新（.valueが必要）
}
</script>

<template>
    <p>カウント: {{ count }}</p>
    <button @click="increment">カウントアップ</button> <!-- イベントバインディング -->
</template>
```

**解説**:
- `ref()`: データが変わったら画面も自動で更新したい場合に使用します。
- `@click`: ボタンが押されたときに関数を実行します。

### 1-2. タイトル変更機能（双方向バインディング）
**課題意図**:
画面からの入力をデータに反映させ、データが変われば画面も変わる「双方向バインディング」を理解する。

**回答コード例**:
```vue
<script setup>
import { ref } from 'vue'

const title = ref('初期タイトル')
</script>

<template>
    <h1>{{ title }}</h1>
    <!-- v-modelでデータと入力欄を同期 -->
    <input type="text" v-model="title"> 
</template>
```

**解説**:
- `v-model`: フォームの入力値と変数を「紐付け」ます。入力すれば変数が変わり、変数が変われば入力欄の中身も変わります。

---

### 1-3. リスト表示機能
**課題意図**:
配列データを扱い、`v-for` を使って一覧表示する。また配列への追加操作を行う。

**回答コード例**:
```vue
<script setup>
import { ref } from 'vue'

const items = ref(['学習課題1', '学習課題2', '学習課題3'])
const newItem = ref('')

const addItem = () => {
    if (newItem.value.trim() !== '') {
        items.value.push(newItem.value)
        newItem.value = '' // 入力欄をクリア
    }
}
</script>

<template>
    <ul>
        <li v-for="item in items" :key="item">{{ item }}</li>
    </ul>
    <input type="text" v-model="newItem">
    <button @click="addItem">追加</button>
</template>
```

**解説**:
- `v-for="item in items"`: 配列の要素を順番に取り出して表示します。
- `items.value.push(...)`: 配列の末尾に新しい要素を追加します。
- `ul` (Unordered List) と `li` (List Item) タグを使って箇条書きリストを作成しました。

### 1-4. リスト削除機能
**課題意図**:
リストの特定の項目を削除する処理を学ぶ。イベントハンドラへの引数渡し (`index`) と配列操作 (`splice` または `filter`) を理解する。

**回答コード例**:
```vue
<script setup>
// ... (他のコードは省略)
const deleteItem = (index) => {
    // index番目から1つ削除する
    items.value.splice(index, 1)
}
</script>

<template>
    <ul>
        <li v-for="(item, index) in items" :key="item">
            {{ item }}
            <!-- indexを引数として渡す -->
            <button @click="deleteItem(index)">削除</button>
        </li>
    </ul>
</template>
```

**解説**:
- `v-for="(item, index) in items"`: ループの「中身」だけでなく「何番目か」の情報も受け取ります。
- `items.value.splice(index, 1)`: 配列操作のメソッド。`index` 番目の要素から `1` 個取り除きます。

### 1-5. 条件付きスタイル適用
**課題意図**:
データの状態に応じて見た目（CSS）を変える「クラスバインディング」を学ぶ。タスク完了状態の切り替えなどに使用します。

**回答コード例**:
```vue
<script setup>
import { ref } from 'vue'

// データ構造をオブジェクトに変更
const items = ref([
    { title: '課題1', isDone: false },
    { title: '課題2', isDone: false }
])

const toggleStatus = (item) => {
    item.isDone = !item.isDone
}
</script>

<template>
    <ul>
        <li v-for="(item, index) in items" :key="item">
            <!-- 条件付きクラス適用 -->
            <span :class="{ done: item.isDone }" @click="item.isDone = !item.isDone">
                {{ item.title }}
            </span>
            <button @click="deleteItem(index)">削除</button>
        </li>
    </ul>
</template>

<style scoped>
.done {
    text-decoration: line-through;
    color: gray;
}
</style>
```

**解説**:
- `:class="{ クラス名: 条件 }"`: 条件が `true` の時だけ、指定したクラスがつきます。
- オブジェクト配列: 1つの項目に対して「タイトル」以外の情報（完了かどうかなど）を持たせたい場合、オブジェクト `{ title: '...', isDone: ... }` を使います。
